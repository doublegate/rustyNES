# Contributing to rustyNES

Thank you for considering contributing to rustyNES! This document outlines the guidelines for contributing to the project.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct:

- Be respectful and inclusive
- Be patient and welcoming
- Be thoughtful
- Be collaborative
- When disagreeing, try to understand why

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue on GitHub with the following information:

- A clear, descriptive title
- A detailed description of the issue
- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Your environment (OS, Rust version, etc.)

### Suggesting Enhancements

Enhancement suggestions are welcome! Please include:

- A clear, descriptive title
- A detailed description of the proposed enhancement
- Any related documentation or examples
- Your reasoning for the enhancement

### Pull Requests

1. Fork the repository
2. Create a new branch (`git checkout -b feature-branch-name`)
3. Make your changes
4. Run the tests to ensure everything works
5. Commit your changes (`git commit -m 'Add some feature'`)
6. Push to the branch (`git push origin feature-branch-name`)
7. Open a Pull Request

#### Pull Request Guidelines

- Follow the existing code style
- Include tests for new features
- Update documentation as necessary
- Keep pull requests focused on a single change
- Link to any relevant issues

## Development Setup

1. Install Rust and Cargo
2. Clone the repository
3. Run `cargo build` to build the project
4. Run `cargo test` to run the tests

## Git Workflow

- Use feature branches for all new features and bug fixes
- Keep a clean commit history with descriptive commit messages
- Rebase your branch before submitting a pull request

## Coding Standards

- Follow Rust's official style guide
- Use `cargo fmt` to format your code
- Use `cargo clippy` to check for common mistakes
- Write doc comments for public API
- Cover your code with tests when possible

## Testing

- Run `cargo test` to run all tests
- Add new tests for new features
- Ensure all tests pass before submitting a pull request

## Documentation

- Keep documentation up to date
- Document public API with doc comments
- Use markdown for documentation files

Thank you for contributing to rustyNES!
